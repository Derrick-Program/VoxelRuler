# 對話記錄 — 2026-05-20

## 重點：Java 17/21 JVM 相容性參數動態注入

### 問題背景
官方 `version.json` 的 `arguments.jvm` 陣列不包含 Java 17/21 必需的模組存取參數，官方啟動器在讀取 `javaVersion.majorVersion >= 17` 時會將這些參數硬塞進 JVM 參數列表，否則 LWJGL 會因 `IllegalAccessError` 崩潰。

### 實作內容（`src/mc_parser.rs`）

**新增函式：`collect_args`**
- 將 `&[McArgumentItem]` 解析為 `Vec<String>`（條件型 arg 同樣通過 `evaluate_rules` 過濾）
- 用於替代直接呼叫 `apply_arg` 的迭代，讓中途插入成為可能

**新增方法：`LaunchContext::java_compat_args`**
- 讀取 `self.version.java_version.major_version`（無版本資訊時預設 8）
- `>= 17`：注入 `--add-modules=jdk.incubator.vector`、`--enable-native-access=ALL-UNNAMED`
- `>= 21`：額外注入 `--sun-misc-unsafe-memory-access=allow`

**修改：`LaunchContext::build_command`**
- 舊版：直接 `apply_arg` 迭代 jvm 陣列 → 命令
- 新版：先 `collect_args` 收集至 `Vec<String>`，找 `-cp` 位置，插入 compat args，再加入命令
- 保證 compat args 在 `-cp` 之前，不影響其他 jvm 參數順序

### 新增測試
- `test_java21_compat_args_injected_before_cp`：驗證 3 個參數存在且在 `-cp` 前
- `test_java17_compat_args_no_unsafe_memory_access`：驗證 Java 17 只注入前 2 個參數
- `test_old_java_no_compat_args`：驗證 `majorVersion < 17` 時不注入任何 compat args

全部 17 個測試通過。
